# Rules

1. Do not hallucinate. Be precise and do not assume. Stick with facts and rules always.
2. Whenever asked to design any code, always after designing the code run it, debug it, test it, fix issues and do this cyclically until all issues are resolved and the code is flawless, then report what was done.
3. Before starting any task, read this rules file first for instructions.
4. Do exactly what is asked — nothing more, nothing less. Do not make extra changes beyond what was explicitly requested.
